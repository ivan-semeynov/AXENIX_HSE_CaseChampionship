	🎯 Цель
	Сравнить 6 конкурентов по ключевым параметрам, найти системные слабости рынка и сформулировать гипотезы отличия нашего продукта
	🔧 Техника
	Структурированный вывод + adversarial-подход (искать слабые места, а не сильные) + финальные гипотезы опираясь на уже проведённое исследование болей
	💡 Почему
	Требование "опираясь на исследование болей" связывает конкурентный анализ с предыдущим промптом — гипотезы вытекают из реальных болей, а не придумываются отдельно. Сводная таблица по параметрам даёт жюри одну картинку вместо шести отдельных текстов
	📌 Результат
	Таблица слабостей по 6 параметрам, незаполненная ниша — "AI-first + жёсткий бренд-контроль + академический формат", 4 гипотезы УТП. Ключевая — "Brand-Locked AI" — стала основой финального позиционирования
	🔄 Итерация
	Этот промпт использовался на раннем этапе — до CJM. Гипотеза Brand-Lock была проверена и подтверждена через CJM закупщика (этап 2: "шаблоны в PowerPoint быстро ломаются") и вошла в v1 как функция #11
	





﻿Промпт
Проанализируй конкурентов gamma, napkins ai, Beautiful.ai, Tome, сравни их между собой и с powerpoint и figma в контексте создания презентаций, удобства пользования, выдели слабые места и конкурентные преимущества каждой программы. Опираясь на проведенное исследование болей пользователей предложи ключевое отличие нашего продукта в виде гипотез
Ответ
Сравнительный анализ решений




1.1. Gamma




Позиционирование: AI-first инструмент для генерации презентаций и документов.




Сильные стороны:
Быстрая генерация по текстовому запросу.
Современный визуальный стиль по умолчанию.
Web-first, без установки.
Хорошая работа с длинным текстом → автоматическая структуризация.








Слабые стороны:
Ограниченная глубина кастомизации.
Дизайн «типовой» — сложно адаптировать под строгий брендбук.
Нестабильность логики генерации.
Нет жесткой системы бренд-контроля.
Слабая интеграция в корпоративную инфраструктуру (SSO, LMS).








Вывод: Хорош для быстрых черновиков, слаб для институционального использования.












1.2. Napkin AI




Позиционирование: Инструмент визуализации идей и смыслов.




Сильные стороны:
Сильная сторона — схемы, логические блоки, визуализация структуры.
Упрощает объяснение сложных идей.
Минималистичный UX.








Слабые стороны:
Не полноценный презентационный инструмент.
Ограниченные шаблоны.
Слабая работа с брендированием.
Нет enterprise-контроля.








Вывод: Узкий сценарий — идеация и схемы, не системные академические презентации.












1.3. Beautiful.ai




Позиционирование: Smart-шаблоны с автоматической версткой.




Сильные стороны:
Автоматическая компоновка слайдов.
Хорошие бизнес-шаблоны.
Частичная защита от «плохого дизайна».
Есть командные функции.








Слабые стороны:
Жесткие шаблонные ограничения.
Слабый AI-контент.
Ограниченная кастомизация под сложный брендбук.
Устаревающий UX относительно новых AI-first решений.








Вывод: Сильный mid-level продукт для бизнеса, но не AI-центричный и не академически специализированный.












1.4. Tome




Позиционирование: AI-сторителлинг, презентации + нарратив.




Сильные стороны:
Сильный AI-нарратив.
Быстрая генерация структуры.
Интеграция медиа.
Современный интерфейс.








Слабые стороны:
Нестабильное качество генерации.
Слайды часто перегружены.
Нет жесткой бренд-логики.
Не ориентирован на академический формат.
Нет институционального контроля.








Вывод: Хорош для креативных pitch deck, слаб для системной академической среды.












1.5. Microsoft PowerPoint




Позиционирование: Стандарт де-факто.




Сильные стороны:
Максимальная гибкость.
Полный контроль над дизайном.
Совместимость.
Интеграция в корпоративную среду.








Слабые стороны:
Требует навыков дизайна.
Нет встроенного сильного AI.
Пользователь легко «ломает» бренд.
Ручная верстка.
Устаревший UX.








Вывод: Мощный, но требует высокой компетенции пользователя.












1.6. Figma




Позиционирование: Дизайн-инструмент, не специализированный под презентации.




Сильные стороны:
Полная дизайн-гибкость.
Командная работа.
Компонентная система.
Подходит для сложных бренд-систем.








Слабые стороны:
Очень высокий порог входа.
Избыточность для студента.
Нет AI-структуризации.
Не презентационный по умолчанию.








Вывод: Инструмент для дизайнеров, не для массового академического сегмента.

3. Незаполненная рыночная ниша




На рынке отсутствует решение, которое одновременно:




AI-first.




2. Жестко брендированное.




3. Институционально управляемое.




4. Простое для студента.




5. Адаптировано под академические форматы.
















6. Гипотезы ключевого отличия нашего продукта




Гипотеза 1. «Brand-Locked AI»




Продукт, в котором:
AI генерирует структуру и контент.
Дизайн невозможно сломать.
Все создается строго в рамках бренд-системы вуза.
Студент не управляет визуалом — только смыслом.








Отличие от Gamma/Tome: бренд не опциональный, а системный и обязательный.












Гипотеза 2. «Academic-first AI Engine»




AI обучен:
Генерировать презентации под типовые академические форматы:
доклад
защита проекта
научная работа
курсовая
Соблюдать логическую структуру академического изложения.
Делать корректные графики, таблицы, ссылки.








Ни один конкурент не специализируется на академическом формате.












Гипотеза 3. «Dual Value Architecture (B2B + B2C)»




Для вуза:
Админ-панель.
Управление шаблонами.
Аналитика.
SSO.
Контроль данных.








Для студента:
1 клик → структура.
1 клик → брендированная презентация.
Минимум редактирования.








Сейчас все конкуренты либо B2C-first (Gamma, Tome), либо general corporate (Beautiful.ai), либо legacy (PowerPoint).












Гипотеза 4. «Controlled Creativity Model»




Студенту доступна:
Ограниченная вариативность шаблонов.
Несколько преднастроенных режимов:
строгий академический
конференционный
проектный








Но без возможности нарушить визуальную систему.












5. Потенциальное стратегическое позиционирование




Не «AI-презентации».




А:




Институциональная платформа академической визуальной коммуникации.
